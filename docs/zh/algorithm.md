# 算法说明

GRASPO 全称是 Group Relative Adaptive Structured Policy Optimization。它可以理解为面向工业界 Agent 结构化输出任务的 GRPO 变体：保留组内相对优势和无 critic 的资源优势，同时把重点放在结构化奖励、自适应采样和困难样本利用上。

GRASPO 不是通用 GRPO 的全面替代品。它更适合答案可以验证的任务，例如 JSON 信息提取、结构化分类、tool call 参数生成、工单解析、日志解析等。对于长 CoT 数学推理或通用偏好对齐，DAPO、GSPO、PPO 等方法可能更合适。

## 核心思想

- 对每个 prompt 生成一组回答。
- 用结构化奖励函数给每条回答打分。
- 在组内计算相对 advantage。
- 首轮已经掌握的样本直接跳过。
- 对困难样本进行自适应重试，寻找有效 reward 方差。
- 丢弃所有 reward 完全相同的无效组。
- 使用裁剪策略目标优化，不需要 critic 模型。

## 设计初衷

工业界 Agent 往往不是只需要“回答得像人”，而是需要稳定地产生可执行、可解析、可审计的结构化结果。典型例子包括：

- 输出严格 JSON
- 生成 tool call 参数
- 从非结构化文本中提取字段
- 对工单、日志、表单进行结构化解析

这些任务的共同特点是：答案通常有 ground truth 或字段级评测方法，格式错误和多余解释会直接影响系统可用性。GRASPO 因此把 reward 设计成“格式 + 内容 + 简洁性”的组合，而不是只看最终答案是否正确。

## 奖励函数

默认奖励由几部分组成：

- 输出结构 marker 奖励
- JSON 合法性奖励
- 字典字段匹配奖励
- 完全匹配 bonus
- 反垃圾文本惩罚

因此 GRASPO 特别适合 JSON 提取、分类、tool call 学习，以及其它可以程序化验证标准答案的任务。

## 与 GRPO 的区别

- GRPO 是更通用的组内相对策略优化方法，常用于数学、代码、可验证推理等任务。
- GRASPO 在 GRPO 思路上增加结构化 reward，显式奖励 JSON/marker/字段匹配，并惩罚无意义文本。
- GRASPO 会跳过首轮已经掌握的样本，避免在明显容易样本上浪费更新。
- GRASPO 会对困难样本进行自适应重试，尽量构造有 reward 方差的训练组。
- GRASPO 会丢弃 reward 全相同的无效组，减少 advantage 为零的无效训练。

这些改动让 GRASPO 更适合结构化 Agent 任务，但也让它更依赖明确的标准答案和任务规则。

## 困难样本 SFT

实践中，GRASPO 可以和困难样本 SFT 配合使用：先用 rollout 找出模型容易失败、但能通过重试获得高分回答的样本，再把这些高质量回答整理成 SFT 数据。这个流程对小数据集尤其有价值，因为它能把 RL 探索到的有效轨迹固化到模型中。
